D. QED's Favorite Permutation
time limit per test
2 seconds
memory limit per test
256 megabytes
input
standard input
output
standard output

QED is given a permutation∗∗ pp of length nn. He also has a string ss of length nn containing only characters LL and RR. QED only likes permutations that are sorted in non-decreasing order. To sort pp, he can select any of the following operations and perform them any number of times:

  • Choose an index ii such that si=Lsi=L. Then, swap pipi and pi−1pi−1. It is guaranteed that s1≠Ls1≠L.
  • Choose an index ii such that si=Rsi=R. Then, swap pipi and pi+1pi+1. It is guaranteed that sn≠Rsn≠R.

He is also given qq queries. In each query, he selects an index ii and changes sisi from LL to RR (or from RR to LL). Note that the changes are persistent.

After each query, he asks you if it is possible to sort pp in non-decreasing order by performing the aforementioned operations any number of times. Note that before answering each query, the permutation pp is reset to its original form.

∗∗A permutation of length nn is an array consisting of nn distinct integers from 11 to nn in arbitrary order. For example, [2,3,1,5,4][2,3,1,5,4] is a permutation, but [1,2,2][1,2,2] is not a permutation (22 appears twice in the array), and [1,3,4][1,3,4] is also not a permutation (n=3n=3 but there is 44 in the array).

Input

The first line contains tt (1≤t≤1041≤t≤104) — the number of test cases.

The first line of each test case contains two integers nn and qq (3≤n≤2⋅1053≤n≤2⋅105, 1≤q≤2⋅1051≤q≤2⋅105) – the length of the permutation and the number of queries.

The following line contains nn integers p1,p2,…,pnp1,p2,…,pn (1≤pi≤n1≤pi≤n, pp is a permutation).

The following line contains nn characters s1s2…sns1s2…sn. It is guaranteed that sisi is either LL or RR, s1=Rs1=R, and sn=Lsn=L.

The following qq lines contain an integer ii (2≤i≤n−12≤i≤n−1), denoting that sisi is changed from LL to RR (or from RR to LL).

It is guaranteed that the sum of nn and qq over all test cases does not exceed 2⋅1052⋅105.

Output

For each query, output "YES" (without quotes) if it is possible, and "NO" (without quotes) otherwise.

You can output "YES" and "NO" in any case (for example, strings "yES", "yes" and "Yes" will be recognized as a positive response).

Note

In the first testcase, s=RRRLLs=RRRLL after the first query. QED may sort pp using the following operations:

  • Initially, p=[1,4,2,5,3]p=[1,4,2,5,3].
  • Select i=2i=2 and swap p2p2 with p3p3. Now, p=[1,2,4,5,3]p=[1,2,4,5,3].
  • Select i=5i=5 and swap p5p5 with p4p4. Now, p=[1,2,4,3,5]p=[1,2,4,3,5].
  • Select i=4i=4 and swap p4p4 with p3p3. Now, p=[1,2,3,4,5]p=[1,2,3,4,5], which is in non-decreasing order.

It can be shown that it is impossible to sort the array after all three updates of the first testcase.
